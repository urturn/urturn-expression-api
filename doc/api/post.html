UT.Post Class

Post instances are retrieved by the callback registered on Expression.ready(callback). They are the entry point of all the functionalities to create a new post on urturn.com

methods: resize, valid, save, on, off, fire, pushNavigation, popNavigation, navigate, collection

properties: context, url, storage, node

Methods

Methods of the Expression object are asynchronous, because ultimately they must communicate with Urturn via the postMessage API.

.dialog( type, [options], [callback] )

Parameters

  • type : A string containing the type of dialog you wish to open. Possible dialogs are 'image', 'sound', 'video', 'crop', and 'text'
  • options [optional] : A hash of options to pass to the dialog. Every type of dialog supports specific options.
  • callback [optional] : A callback function that will be called with results of dialog.
    (Warning ! Some dialog return null when users quit them without performing actions)

Usage

The dialog method lets you display a dialog to the user. Different types of dialogs are supported and each of them supports a different set of options. The result of the dialog operations will be passed through the callback. list of supported dialogs :

  • 'image' : Ask the user to select an image (more details)
  • 'sound' : Ask the user to select a sound
  • 'video' : Ask the user to select a video
  • 'crop' : Diplay a panel that lets users crop an image
  • 'text' : Diplay a panel that lets users enter a text (more details)

Code example

  UT.Expression.ready(function(post) {
    post.dialog('image', {
      size : {width : 200, height : 200}},
      function(image) {
        if (!image) {
          alert ('You select no images');
        }
        else {
          alert ('You select : ' , img.url);
        }
    })
  });

.resize( sizeInfo )

Parameters

  • sizeInfo : The size at wich you want to resize the post. Possible values are 'auto', {height: 99}, {height: '99px'}, 99

Usage

Ask the container to resize to the given parameters. The asynchronous result of this function can be listened on the post resize event.
Currently just support resize of post height.

Code example

  UT.Expression.ready(function(post) {
    // Listen to the resize event
    post.on('resize', function() {
      alert('This post height has been changed');
    });
    // set post height to 500px
    post.resize({height : '500px'});
  });

.getUserData(callback)

Parameters

  • callback : The parameters of the function called with userdatas are defined once they are retrieved

Usage

Return current user Metadata in callback.
Value passed as callback parameter : 

{
  username : 'John',
  avatar   : 'http://linktourturnavatar.com',
  numberOfPost : 116, // [Edit mode only] the number of posts made by a user
  uuid : 'uuid:of:current:user', // [Edit mode only] 
  numberOfUse : 12 // [Edit mode only] number of times this user uses this Expression
}

Code example

UT.Expression.ready(function(post) {
  post.getUserData(function (data) {
    alert('hello ' + data.username + '!');
  });
});

.valid(flag)

Parameters

  • flag (boolean) : True if the post is valid and can be posted, false if not.

Usage

By default a user can't publish a post on Urturn until you set it to valid. Once you have set a post as valid it can be saved synchronously at any time.
You can set back a post to invalid to post using false as parameter.
When the post is vaild the user sees the post button, when it's not, the post button is hidden or unclickable (mobile).

Code example

UT.Expression.ready(function(post) {
  // User can post
  post.valid(true);
  // User can't post
  post.valid(false);
});

.save()

Usage

Enables you to save the content of the post to the server.

Code example

UT.Expression.ready(function(post) {
  // set some value inside storage
  post.storage.day = 'sunday';
  post.storage.rainny = true;
  // save storage on serveur
  post.save();
});

.on(eventName, callback)

Parameters

  • eventName (string) : The name of the event to listen ('publish', 'scroll', 'resize', 'media').
  • callback : A callback function called when event is fire.

Usage

Enables you to listen various events inside your Expression ('publish', 'scroll', 'resize', 'media'). The callback will be passed arguments related to the event.

Code example

UT.Expression.ready(function(post) {
  post.on('scroll', function () {
    alert('Post was scrolled');
  })
});

.off(eventName, callback)

Parameters

  • eventName (string) : The name of the event to stop listen ('publish', 'scroll', 'resize', 'media').
  • callback : A callback function to remove from list of listener.

Usage

Remove the callback function from the list of registered listener for the given eventName.

Code example

UT.Expression.ready(function(post) {
  function  scrolled() {
    alert('Post was scrolled');
  }
  // add an event
  post.on('scroll', scrolled);
  
  // remove an event
  post.off('scroll', scrolled);
});

.pushNavigation(state, callback)

Parameters

  • state (string) : A navigation state ('default', 'cancel' or 'back')
  • callback : A callback function called when the state is pop by user navigation.

Usage

Pushnavigation enables you to push a navigation state to override Urturn UI. This will change the navigation button. The different possible states are :

  • default : the default state. When a user clicks on the navigation button, he leaves the editor
  • cancel : display a cancel button in place of the standard button. When a user clicks on it, the callback is called and this state is pop (this means the button goes back to its previous state).
  • back : display a back button in place of the standard button. When a user clicks on it, the callback is called and this state is pop (this means the button goes back to its previous state).

Code example

UT.Expression.ready(function(post) {
  post.pushNavigation('cancel', function() {
    alert('User push cancel button');
  })
});

.popNavigation()

Usage

popNavigation pop the active pop without calling the attached callback.

Code example

UT.Expression.ready(function(post) {
  post.pushNavigation('cancel', function() {
    alert('User push cancel button');
  });
  // pop the cancel button and display the default one.
  // does not call attached callback
  post.popNavigation();
});

.navigate([app], target)

Parameters

  • app (optional, default=browse) : The 'app' to use to navigate ('browse', 'search', 'user')
  • target (string): The navigation target

Usage

navigate lets you navigate inside and outside of Urturn. There are three different navigation 'apps'

  • browse : browse a website (ex : example.org)
  • search : do a search on Urturn and display the result page.
  • user : display a user profile on Urturn.

Code example

UT.Expression.ready(function(post) {
  // take user to #rocks search result page
  post.navigate('search', '#rocks');
});

.collection(name)

Arguments

  • name (string) : Name of the collection to retrieve.

Return

An UT.PublicCollection instance or an UT.Collection instance.

Usage

Retrieve a storage collection given its name.

The name can be:

  • any collection defined in expression.json,
  • 'parent' to retrieve the storage item from the parent document.
  • 'default' to retrieve the default collection (aka Post.storage)

Code example

UT.Expression.ready(function(post) {
  // Get the parent collection
  post.collection('parent');
});

Properties

.context [Read ONLY]

Usage

{
  player: boolean, // True if in player mode
  editor: boolean, // True if in editor mode
  privacy: String, // One of 'private', 'unlisted' or 'public', the current state of the document publication
}

This property should not be modified as the context is read-only.

.node

the Expression outter DOM node

.storage

Storage is the place where you store your data in an Expression. It is an instance of UT.Collection

.url

Retrieve the public url of the document.

.note

The 'note' of the document.

The note is the text display under the post and use when a post is shared on social networks. By default the note is empty. When an user creates a post, he will have the opportunity to change the note. The note is accessible inside the Expression using the post.note property. In edit mode, you have the possibility to modify it :

UT.Expression.ready(function(post) {
  post.note = 'This is a new note!';
})
